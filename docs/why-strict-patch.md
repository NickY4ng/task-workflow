# 为什么严格 Patch？

## 问题：AI 改多了

用户说："把 A 改成 B"

AI 实际：改了 A，顺手把 C、D、E 也改了

用户："我只让你改 A！"

---

## 原因：AI 的"过度热心"

AI 会：
- "我觉得这里也不太对，一起改了吧"
- "这样改更一致"
- "用户可能也想要这个"

**结果：** 改多了，改错了，用户更不满意。

---

## 方案：严格 Patch

### 规则

**用户说改 A → 只改 A**

- 匹配用户明确的 old_string
- 替换为 new_string
- 其他内容不动

### 校验

```python
count = content.count(old_string)
if count == 0:
    return "找不到匹配文本"
if count > 1:
    return "匹配不唯一，请提供更精确的上下文"
```

### 禁止

- 不扩展修改范围
- 不猜测用户意图
- 不改未提及内容

---

## 效果

**传统方式：**
```
用户：改 A
AI：改了 A，顺便改了 B、C
用户：B 我没让改！
AI：抱歉，改回来...
（又多了一轮）
```

**严格 Patch：**
```
用户：改 A
AI：只改了 A，请确认
用户：OK
（一轮结束）
```

---

## 核心原则

**听话，不多做。**

用户是导演，AI 是演员。
导演说演哪场，就演哪场，不擅自加戏。
