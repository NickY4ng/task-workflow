# Task Workflow V3

## 五步状态机任务管理框架

**解决 Agent 产出与用户预期不一致的问题**

---

## 核心问题：Verification Burden

AI 直接生成 → 人逐字核查 → 反复修改 → 效率低下

**痛点：**
- 产出和预期不一致
- 反复确认消耗大量时间
- 上下文丢失导致重复劳动

---

## 解决方案：五步状态机

```
Step 1: 复述确认 → 先对齐理解，再动手
Step 2: 方案设计 → 确认怎么做，再执行
Step 3: 执行     → 按方案做，不偏离
Step 4: 自检     → 交付前自验
Step 5: 交付     → 确认完成
```

**硬约束：** 不可跳过步骤，必须确认

---

## 关键机制

### 1. 状态机硬约束
- 每步必须确认才能推进
- 用户说"改"→回退上一步
- 用户说"结束"→跳出任务模式

### 2. 事件驱动管理
- 每个任务有唯一 ID
- 状态全程可追溯
- 多任务并行支持

### 3. 状态记录
- 过程日志自动记录
- 上下文不丢失
- 可恢复、可审计

### 4. 严格 Patch
- 只改用户明确说的部分
- 不动未提及内容
- 避免猜测式修改

### 5. 自动交付
- 文档输出后自动发送
- 不遗漏、不等待

---

## 案例研究

### 背景
某企业数据平台项目，需求复杂（10+筛选条件、16个输出字段），传统方式反复修改 5-6 版。

### 应用五步状态机后
- Step 1: 复述确认需求范围
- Step 2: 设计数据下载+解密方案
- Step 3: 按方案输出文档
- Step 4: 自检字段和流程
- Step 5: 交付确认

**结果：** 一次性到位，减少 80% 反复修改
