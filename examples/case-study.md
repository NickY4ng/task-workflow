# 案例研究：数据平台项目

## 背景

某企业客户需要一套数据下载和查询系统，涉及：
- 3 类数据下载（新增/保有量/迁出）
- 10+ 筛选条件
- 16 个输出字段
- 企业名称加密处理

## 挑战

### 传统方式的问题

1. **需求复杂**：筛选条件多，容易遗漏
2. **字段敏感**：企业名称需要掩码处理
3. **反复修改**：输出格式调整 5-6 版
4. **上下文丢失**：改着改着忘了最初要求

## 应用五步状态机

### Step 1: 复述确认

```
AI: 我的理解是：
- 核心任务：数据平台方案设计
- 功能：数据下载 + 数据解密
- 输出：CSV 格式
- 企业名称：掩码处理

方向对不？

用户: 对
```

### Step 2: 方案设计

```
AI: 方案如下：
- 模块1：数据下载（3类数据）
- 模块2：数据解密（一次一个）
- 交互：对话框确认条件
- 输出：CSV 下载

确认吗？

用户: 确认，但解密要持续交互，不禁锢
```

### Step 3: 执行

按方案输出文档，每步只改用户说的部分。

### Step 4: 自检

- 字段是否完整？
- 流程是否正确？
- 敏感信息是否规避？

### Step 5: 交付

用户确认，一次性到位。

## 效果

| 指标 | 传统方式 | 五步状态机 |
|------|---------|-----------|
| 修改轮次 | 5-6 轮 | 1-2 轮 |
| 确认时间 | 分散在各轮 | 集中在 Step 1-2 |
| 上下文丢失 | 经常 | 无（事件记录） |
| 最终满意度 | 勉强可用 | 一次性到位 |

## 关键经验

1. **复述确认**：确保理解一致，避免方向性错误
2. **方案设计**：确认后再执行，减少中途变更
3. **严格 patch**：只改用户说的，不擅自扩展
4. **事件记录**：全程可追溯，不丢失上下文

## 脱敏说明

本案例基于真实项目经验，已脱敏处理：
- 企业名称：用"某企业客户"代替
- 产品名称：用"数据平台"代替
- 具体字段：用通用描述代替
